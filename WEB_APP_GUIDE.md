# Streamlit Web 应用启动指南

## 快速开始

### 方式1：直接运行（推荐）

```bash
# 启动 Web 应用
streamlit run web_app.py
```

### 方式2：使用启动脚本

```bash
# Mac/Linux
./run_web.sh

# Windows
python web_app.py
```

## 访问应用

启动后，在浏览器中打开：
```
http://localhost:8501
```

## 功能说明

### 侧边栏功能
- **📖 简介**: 系统功能介绍
- **🔑 配置**:
  - 使用模拟层（Mock）: 无需API密钥，快速测试
  - DeepSeek API Key: 生产环境使用真实API
- **🔄 重新加载知识库**: 重新加载data目录下的文档
- **📊 系统状态**: 显示文档数量和索引状态

### 主界面
- **顶部标题**: "垂直领域专家助手"
- **聊天区域**: 显示对话历史，区分用户和助手消息
- **输入框**: 底部固定，输入问题并获取回答

## 配置选项

### 使用模拟层
勾选侧边栏的"使用模拟层"选项，无需API密钥即可测试系统功能。

### 使用真实API
1. 在 https://platform.deepseek.com/ 获取 API Key
2. 在侧边栏输入 API Key
3. 取消勾选"使用模拟层"

### 添加知识库文档
1. 将 `.txt` 或 `.md` 文件放入 `data/` 目录
2. 点击侧边栏的"重新加载知识库"按钮
3. 等待加载完成

## 技术特性

- ✅ 自动加载和索引知识库
- ✅ Top-K=3 相似度检索
- ✅ 流式输出支持
- ✅ 多轮对话支持
- ✅ 响应式设计
- ✅ 模拟层和真实API无缝切换

## 注意事项

1. **首次运行**: 需要下载嵌入模型，可能需要几分钟
2. **大文件处理**: 建议将大文档拆分成小文件
3. **API配额**: 使用真实API时注意调用频率限制
4. **内存使用**: 知识库较大时可能占用较多内存

## 故障排除

### 问题：无法加载模型
**解决方案**: 检查网络连接，确保能够访问 HuggingFace

### 问题：索引创建失败
**解决方案**: 确保 `data/` 目录下有有效的文档文件

### 问题：API调用失败
**解决方案**: 检查API密钥是否正确，或切换到模拟层

### 问题：端口占用
**解决方案**: 使用其他端口启动
```bash
streamlit run web_app.py --server.port 8502
```

## 性能优化

### 提升检索速度
- 减少文档数量
- 降低 Top-K 值
- 使用更小的嵌入模型

### 提升响应速度
- 使用 GPU 加速（修改 `device="cuda"`）
- 减少上下文长度
- 优化 Prompt 模板

## 开发模式

启用自动重载：
```bash
streamlit run web_app.py --server.runOnSave true
```

## 生产部署

使用 Streamlit Cloud 部署：
1. 将代码推送到 GitHub
2. 在 Streamlit Cloud 中导入仓库
3. 配置环境变量（DEEPSEEK_API_KEY）
4. 自动部署
