# Web 应用修复说明

## 修复内容

### 1. 侧边栏控件重复创建问题

#### 问题
- 原代码在 `render_sidebar()` 和 `main()` 中都创建了控件
- 导致"使用模拟层"复选框和"DeepSeek API Key"输入框被重复创建
- 用户修改设置时出现两个控件不同步的问题

#### 解决方案
- 在 `render_sidebar()` 中使用固定的 `key` 参数确保控件只创建一次
```python
use_mock = st.checkbox("使用模拟层 (Mock)", key="use_mock_main", ...)
api_key = st.text_input("DeepSeek API Key", key="api_key_main", ...)
```
- 将配置保存到 `st.session_state`，其他函数从 session state 读取
```python
st.session_state.use_mock = use_mock
st.session_state.api_key = api_key
```
- 在 `main()` 中移除了重复的控件创建代码

### 2. 代码执行顺序优化

#### 问题
- 原代码在加载知识库时，侧边栏的状态显示不实时
- 文档数量和索引状态没有即时更新
- 用户体验不佳

#### 解决方案
重新组织了 `main()` 函数的执行顺序：

```python
def main():
    # 1️⃣ 初始化所有 session state
    # 2️⃣ 渲染侧边栏（获取配置）
    # 3️⃣ 从 session state 读取配置
    # 4️⃣ 初始化模型（按需）
    # 5️⃣ 初始化LLM（按需）
    # 6️⃣ 加载文档和创建索引（按需）
    #    - 实时更新 document_count
    #    - 实时更新 index_status
    #    - 显示加载进度
    # 7️⃣ 渲染主界面内容
    # 8️⃣ 渲染聊天历史和输入
```

### 3. 状态同步优化

#### 新增功能
1. **加载进度指示**
```python
if "loading_progress" in st.session_state and st.session_state.loading_progress:
    st.info("🔄 正在加载知识库...")
```

2. **实时文档数量更新**
```python
st.session_state.document_count = len(documents)
```

3. **索引状态实时同步**
```python
st.session_state.index_status = "就绪" / "无文档" / "未就绪" / "加载失败"
```

4. **重新加载后自动刷新**
```python
if st.button("🔄 重新加载知识库", ...):
    st.session_state.reload_index = True
    st.rerun()  # 确保状态更新
```

### 常见问题

**Q: 为什么还是看到重复的控件？**
A: 清除浏览器缓存或重启应用，session state 可能包含旧数据。

**Q: 状态更新不及时怎么办？**
A: 在关键位置添加 `st.rerun()` 确保界面刷新。

**Q: 如何调试状态问题？**
A: 使用 `st.write(st.session_state)` 查看当前状态。
